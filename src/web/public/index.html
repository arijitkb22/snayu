Connected Agents

Register external AI agents so they can communicate with each other through the connector.

How It Works

Register your specialized agents (e.g. Jira Analysis Agent, Defect Triage Agent) here. Each agent gets a unique ID and endpoint. Once registered, agents can:

  • Discover other registered agents via connector_list_agents
  • Send messages to other agents via connector_send_to_agent
  • Share context and delegate tasks across agent boundaries

Active Tools

Tools currently exposed to your AI agents via the Snayu MCP — includes both service connections and onboarded agents.

Agent Registry

Register your agent here — define its tools and endpoint. Once registered, those tools are instantly available to all users via Snayu MCP with zero extra setup on their end.

How It Works

For developers: Register your agent with a manifest (name, endpoint, tools). Snayu validates it, runs a health check, and namespaces your tools to avoid collisions.

For users: New tools appear automatically in every connected AI agent (Copilot, Claude, Cursor…) — no config changes, no restarts. They just work.

// Your agent exposes tools via HTTP
POST /tools/analyze_ticket → { ticketId: "PROJ-123" }
POST /tools/find_duplicates → { ticketId: "PROJ-123", threshold: 0.8 }

// After onboarding, ALL Snayu users get:
ragent_xxx__analyze_ticket
ragent_xxx__find_duplicates
